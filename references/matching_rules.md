# 评论 → 模板匹配启发规则

本文给 Claude 看，目的是让模板匹配稳定可预测。不要把这里的关键词当 hard rule —— 语义优先于词面。

## 1. 类别大区映射（先粗分）

读到评论文本时，先归类到大区，再在该大区内挑具体模板：

| 大区 | 关键词信号（任意语言） | 典型类别 |
|------|----------------------|----------|
| **广告系** | ad / 广告 / реклама / publicidad / annonce / werbung / pubblicità | 广告太多 / 广告无法关闭 / 离线广告 / 诱导下载广告 / 低俗广告 / 广告太长了 / 禁用广告 |
| **权限授权系** | permission / authorize / access denied / 权限 / разрешение / Android 13/14 | Android13 / 14 系统授权引导 / Android data 授权 / sd 卡重新授权 |
| **购买/付款系** | purchase / restore / VIP / premium / 订阅 / 付费 / 价格 | 恢复购买 / 无法识别购买 / 抱怨收费高 / 价格不一致 / 在多个设备上识别购买 |
| **崩溃/Bug 系** | crash / freeze / not working / 崩溃 / 闪退 / bug | 确实有 bug / 问题已在下版本修复 / 截图或录屏 |
| **功能咨询** | how to / where is / can't find / 怎么 / 如何 / 找不到 | 询问详细问题 / 如何使用 / 找不到转换的文件 |
| **建议** | suggest / hope you add / 建议 / 希望 | 建议 / 询问详细建议 / 抱怨没有这个功能 |
| **空泛反馈** | "good" / "bad" / "不好用" / 单纯 1 星无内容 / 单纯 5 星无内容 | 无意义评论 / 无意义差评 / 要五星 |

## 2. 短评/空泛评论：先看「文字情感」，再看星级

短评（没有具体诉求、就是个态度表态）的匹配关键是**文字本身的情感** + **星级落差**，**不要只看星级**：

| 文字情感 | 星级 | 命中类别 | 理由 |
|---------|------|---------|------|
| **正面 / 肯定**（good / nice / cool / great / love it / ok / okay / fine / 好 / 好的 / 不错 / 嗯 / 👍 / 极短正面 / 无文字但语气正面） | **1-4★** | **求五星类**：XFolder→`要五星`；MP3 Cutter / Video to MP3→`无意义评论求五星`（或 `说好但没有五星`） | 用户态度正面却没给满星 = 引导补五星的最佳时机 |
| 正面 / 肯定 | 5★ | 求五星类（求保持五星 / `要五星`） | 已满星，致谢 + 维稳 |
| **负面 / 贬损**（bad / shit / useless / terrible / 垃圾 / 难用，或纯空 1-2★ 无任何正面信号） | 1-2★ | `无意义差评` | 态度负面但没给细节，礼貌请其补充 |
| 表扬带"差一点 / 还差点啥" | 4★ | `缺点+建议没给五星` | |
| 完全 emoji / 重复字符 / 乱码，判不出情感 | 任意 | `不理解的评论` | |

**判定要点（重要，别走回头路）：**

- 「ok / okay / 好的 / 嗯」这类**中性偏肯定**的词，在 1-4★ 下一律按**正面**处理 → 走求五星类。只有评论里出现**明确负面词**或**具体抱怨内容**时才走 `无意义差评`。即使是 1★，只要文字本身是肯定/中性（如 "Ok"、"好"），也走求五星类——星级和文字矛盾时，文字正面就是争取五星的机会。
- `无意义评论`（XFolder `xfolder-004`、问"喜欢或不喜欢什么"）仅在**情感真正中性、星级也中性（约 3★）、确实看不出正负**的空泛评论才用；**正面 + 1-4★ 不要再命中它**，要命中求五星类。
- 上述「正面短评 + 1-4★ → 求五星类」是**高置信命中（confidence ≥ 0.9）**，路径 A 应直接填入，不要因为"看起来空泛"就判 unmatched。

## 3. 多语言注意

- 评论可能是非英文。`reviewer_language` 字段给的是 ISO 代码（如 "ru"、"pt"），优先信任它；为空时根据文本判断。
- **不要**根据评论语言决定回复语言 —— 回复语言由 `target_language` 参数指定。
- 匹配类别时用语义，不要被翻译过的 `text`（中文译文）字面误导。优先看 `original_text`（用户真实输入）。

## 4. 置信度评估口径

输出每个候选时给一个 `confidence` ∈ [0, 1]，含义：

- **0.85+**：评论意图与类别完全吻合，模板可直接发出。
- **0.6 - 0.85**：意图大致吻合，但模板里有部分内容（版本号、功能名）需要 LLM 微调或不太合身。
- **0.3 - 0.6**：勉强相关，更像兜底（如"无意义差评"对所有 1 星短评）。
- **< 0.3**：不建议使用，仅作为陪跑候选。

如果**没有任何模板置信度 ≥ 0.3**，路径 A 输出时把 `candidates` 里只放 1 条 `source: "generated"` 的原创回复，不强凑模板。

## 5. 必须避免的坑

1. **不要**对同一条评论同时返回 3 条都是"无意义差评/广告太多"等兜底类别 —— 候选间要有意义的差异。如果实在找不到 3 条不同模板，就少返回，宁缺勿滥。
2. **不要**为 "ringwall" / "xplayer" 这两个无模板产品强行套用 MP3 Cutter 或其他 sheet 的模板。它们应直接走 `source: "generated"`，并在 `meta.note` 里标"该产品暂无模板"。
3. **不要**编造模板里没有的版本号 / 用户名 / 邮箱。模板里出现的占位（如 "version 1.4.8" / "filemanager.feedback@gmail.com"）原样保留；若评论里能明确版本，可保留模板原版本号或替换为 `app_version_name` 字段值，但不要瞎填。
4. **不要**在 `confidence` 上撒谎来强行让模板被选中。低就是低。
